# Online Known Issues

オンライン対戦まわりで、現時点で確認できている問題をまとめる。

## 0. ローカルでのテスト方法

ローカル（`file://` や `localhost`）でも Firebase に接続できるため、2ブラウザタブを開けばオンライン対戦のテストが可能。

**注意:** 同じブラウザの2タブだと匿名認証 UID が共有されるが、`clientId`（タブ単位のランダムID）で更新を区別しているため問題なし。

## 1. 公開ルーム一覧に公開ルームが表示されない ← 要対応

### 原因

Firebase Realtime Database の Rules で `/rooms` に対する読み取り権限がない。

### 解決手順

Firebase Console（https://console.firebase.google.com）から以下を設定する：

1. プロジェクト「hanahuda-game」を開く
2. 左メニュー → **Realtime Database** → **ルール** タブ
3. 以下の内容に書き換えて「公開」ボタンを押す：

```json
{
  "rules": {
    "rooms": {
      ".read": true,
      "$roomId": {
        ".write": "auth != null"
      }
    }
  }
}
```

→ `database.rules.json` にも同内容を保存済み（参照用）。

### Anonymous Auth が有効になっているか確認

1. Firebase Console → **Authentication** → **Sign-in method**
2. 「匿名」が有効になっているか確認・有効化する

## 2. オンライン対戦の同期がまだ不安定な可能性がある

### 対応状況

- `updatedBy` 判定を Firebase UID からタブ単位の `clientId` に変更済み
- これにより、同一ブラウザ2タブでも更新を捨てないよう修正済み

### 残チェック項目

- こいこいで初期盤面が両画面に出るか
- 1ターン進行後も手札・場・ターン表示が一致するか

---

## 3. 切断時の終了処理は再検証が必要

### 対応状況

- ゲスト側の `players/1` に `onDisconnect().remove()` を追加済み
- 相手プレイヤーが消えたらセットアップ画面へ戻す処理を追加済み

### 残チェック項目

- タブを閉じたときに相手側が戻るか
- 「戻る」ボタンで離脱したときに相手側へ反映されるか
- ホスト離脱時にルーム全体が削除されるか

役が揃った際の「こいこい」「勝負」の選択がオンライン側へ即時反映されるか要確認。
