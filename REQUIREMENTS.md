## 1. プロジェクト概要

### 1.1 コンセプト

花札の札・月・役の体系をそのまま保持しつつ、
トランプの**神経衰弱**のゲーム性を組み合わせた新しいカードゲーム。

「同じ月の札を揃えて回収する」という神経衰弱の記憶要素に加え、
「どの札を取るか」という花札の役づくり戦略が絡み合う。

### 1.2 制作意図・ポートフォリオとしての位置づけ

- 日本の伝統的なカードゲームをデジタル化し、現代のブラウザで手軽に遊べる形にする
- 花札を知らないプレイヤーが神経衰弱から入り、自然と花札の役・月・植物を覚えられる設計
- シングルHTMLファイルで完結する実装力のデモンストレーションとしても活用する
- 自前サイト（ポートフォリオ）＋ itch.io（ゲーム公開）の二段構えで展開する

### 1.3 対象プレイヤー

| ターゲット | 詳細 |
|-----------|------|
| メイン | 花札に興味はあるが詳しく知らない20〜40代 |
| サブ | 花札を知っている人が新鮮なアレンジとして楽しむ |
| 対象人数 | 2〜4人（同一デバイスでのパスプレイ） |

## 2. 花札の基礎知識

> ゲーム内のチュートリアルや説明画面に組み込む内容。
> 花札を知らないプレイヤーでも理解できるよう記述する。

### 2.1 花札とは

花札は**48枚のカード**（12ヶ月 × 4枚）で構成される日本の伝統的なカードゲーム。
各月には季節の花や植物が描かれており、それぞれの月に4種類のカードがある。
カードの組み合わせで「役（やく）」を作り、ポイントを競う。

### 2.2 カードの種類

| 種別 | 読み | 特徴 | 点数（基本） |
|------|------|------|-------------|
| 光 | ひかり | 最も格上のカード。鶴・月・桜の幕など豪華な絵柄 | 20点 |
| 種 | たね | 動物や道具が描かれた中級カード | 10点 |
| 短冊 | たんざく | 短冊（細長い紙）が描かれたカード。赤・青の2種 | 5点 |
| カス | かす | 絵柄がシンプルなカード。枚数が最も多い | 1点 |

> 本作では個別カードの点数ではなく、**集めたカードで作れる「役」の合計点**で勝敗を決める。

### 2.3 12ヶ月と植物の対応

| 月 | 植物 | 読み | 代表カード |
|----|------|------|-----------|
| 1月 | 松 | まつ | 松に鶴（光） |
| 2月 | 梅 | うめ | 梅に鶯（種） |
| 3月 | 桜 | さくら | 桜に幕（光）|
| 4月 | 藤 | ふじ | 藤に不如帰（種） |
| 5月 | 菖蒲 | あやめ | 菖蒲に八橋（種） |
| 6月 | 牡丹 | ぼたん | 牡丹に蝶（種） |
| 7月 | 萩 | はぎ | 萩に猪（種） |
| 8月 | 芒 | すすき | 芒に月（光） |
| 9月 | 菊 | きく | 菊に盃（種） |
| 10月 | 紅葉 | もみじ | 紅葉に鹿（種） |
| 11月 | 柳 | やなぎ | 柳に小野道風（光・雨） |
| 12月 | 桐 | きり | 桐に鳳凰（光） |

### 2.4 代表的な花札の遊び方（参考）

本作は花札の「神経衰弱アレンジ版」だが、元となる代表的な遊び方も記録しておく。

#### こいこい（最も有名な2人用遊び）

| 項目 | 内容 |
|------|------|
| 人数 | 2人 |
| 目的 | 手札から場の同月カードを取り合い、役を完成させて点数を競う |
| 特徴 | 役が完成したとき「こいこい（継続）」か「勝負（終了）」を選択する駆け引きがある |
| 終了 | 手札がなくなったら1ラウンド終了。12ラウンド（12ヶ月分）行うことが多い |

#### 花合わせ（3〜4人向け）

| 項目 | 内容 |
|------|------|
| 人数 | 3〜4人 |
| 目的 | こいこいと似た方式で、複数人で役を競う |
| 特徴 | 人数が多い分、駆け引きがより複雑になる |

#### 神経衰弱（本作のベース）

| 項目 | 内容 |
|------|------|
| 人数 | 2人〜 |
| 目的 | 裏向きの札から同じ月を2枚揃えて回収する |
| 特徴 | 最もシンプルで、花札の絵柄を覚えるのに最適。子供でも遊べる |
| 難易度 | ★☆☆（初心者向け） |

> 難易度の目安：神経衰弱 ★☆☆ → こいこい ★★☆ → 花合わせ ★★☆ → 六百間 ★★★

## 3. ゲームルール（本作）

### 3.1 基本ルール

| 項目 | 内容 |
|------|------|
| 使用札 | 花札48枚（12ヶ月 × 4枚） |
| プレイヤー数 | 2〜4人 |
| 初期配置 | 全48枚を裏向きでグリッド状（8列×6行）に並べる |
| ターン | 決めた順番に従いプレイヤーが交代 |
| 1ターンの行動 | 好きな2枚をめくる |
| 成功条件 | めくった2枚が**同じ月** → 回収（手札に加える） |
| 失敗条件 | 異なる月 → 裏向きに戻してターン終了 |
| ゲーム終了 | 場の札がすべてなくなった時点 |
| 勝敗判定 | 回収した札で作れる**役のポイント合計**が高い方が勝ち |

### 3.2 神経衰弱との違い・本作のオリジナル要素

| 比較項目 | 通常の神経衰弱 | 本作 |
|----------|--------------|------|
| カード | トランプ（同マーク・同数字） | 花札（同じ月） |
| 勝敗基準 | 取得枚数が多い方 | 集めた役のポイント合計 |
| 戦略性 | 記憶力のみ | 記憶力＋役づくりの判断 |
| 1月のカード数 | 2枚（ペア） | 4枚（2ペア分） |

### 3.3 ターンボーナス

- **現バージョン**：一致しても**ボーナスターンなし**（次のプレイヤーへ移行）
- **拡張オプション（未実装）**：一致時にもう1ターン獲得

### 3.4 同月4枚の扱い

各月4枚あるため、1つの月の札は2回のペアマッチで完全回収される。
同月の「光」と「カス」がある場合、どちらのペアを取れるかで役の成立が大きく変わる。
→ **記憶力 × 役の優先順位**が本作の戦略的コアになる。

## 5. 役定義

### 5.1 光系役（排他：最高位のみ成立）

| 役名 | 条件 | 点数 |
|------|------|------|
| 五光 | 光札5枚すべて | 15点 |
| 四光 | 雨男（11月光）を除く光4枚 | 8点 |
| 雨四光 | 雨男（11月光）を含む光4枚 | 7点 |
| 三光 | 雨男を除く光3枚 | 5点 |

### 5.2 特殊役（複数同時成立可）

| 役名 | 必要カード | 点数 |
|------|-----------|------|
| 猪鹿蝶 | 萩に猪（7月種）＋紅葉に鹿（10月種）＋牡丹に蝶（6月種） | 5点 |
| 赤短 | 1月・2月・3月の赤短冊3枚 | 5点 |
| 青短 | 6月・9月・10月の青短冊3枚 | 5点 |
| 月見で一杯 | 芒に月（8月光）＋菊に盃（9月種） | 5点 |
| 花見で一杯 | 桜に幕（3月光）＋菊に盃（9月種） | 5点 |

> 「菊に盃」は月見・花見の両役で共有できる（両方同時成立 = 10点）。

### 5.3 数え役（累積加算）

| 役名 | 条件 | 点数計算 | 例 |
|------|------|----------|-----|
| 種 | 種札5枚以上 | 1点 + (枚数−5)点 | 7枚 → 3点 |
| 短冊 | 短冊5枚以上（赤・青問わず） | 1点 + (枚数−5)点 | 6枚 → 2点 |
| カス | カス札10枚以上 | 1点 + (枚数−10)点 | 12枚 → 3点 |

## 6. UI・画面構成

### 6.1 画面遷移

```
セットアップ画面
  └──[ゲーム開始]──→ ゲーム画面
                          └──[場が空]──→ 結果画面
                                            └──[もう一度遊ぶ]──→ セットアップ画面
```

### 6.2 セットアップ画面

- プレイヤー人数の選択（2〜4人）
- 各プレイヤー名の入力（デフォルト：「プレイヤーN」）
- 遊び方の簡単な説明テキスト
- ゲーム開始ボタン
- （未実装）役一覧・チュートリアルへのリンク

### 6.3 ゲーム画面

- **プレイヤーバー**：名前・取得枚数・現在ポイントを常時表示、アクティブプレイヤーをハイライト
- **ステータスメッセージ**：ターン案内・マッチ結果・ミス通知
- **カードグリッド（8列×6行）**：全48枚を均一配置。回収済みは非表示（位置は保持）

### 6.4 結果画面

- 1〜4位の順位表示（点数降順 → 同点時は取得枚数で比較）
- 各プレイヤーの成立役一覧と点数内訳
- 役なし → 「役なし（0点）」と明示
- 「もう一度遊ぶ」ボタン

## 8. アニメーション仕様

### 8.1 実装済み

| アニメーション | 実装方式 | 時間 | 内容 |
|----------------|----------|------|------|
| カードフリップ | CSS 3D `rotateY` + `preserve-3d` | 450ms | 裏→表・表→裏の3D回転 |
| マッチグロー | CSS `@keyframes` + `box-shadow` | 600ms | 一致時に金色に発光 |

#### CSS実装（カードフリップ）

```css
.card-inner {
  transform-style: preserve-3d;
  transition: transform 0.45s ease;
}
.card.flipped .card-inner {
  transform: rotateY(180deg);
}
.card-back,
.card-face {
  -webkit-backface-visibility: hidden; /* Safari対応必須 */
  backface-visibility: hidden;
}
```

### 8.2 未実装（優先度順）

| 優先度 | アニメーション | 内容 | 実装方針 |
|--------|----------------|------|----------|
| 高 | **ミスシェイク** | 不一致時に2枚がぶるっと左右に揺れる | `@keyframes shake` で `translateX` を振動 |
| 高 | **ターン交代オーバーレイ** | 次プレイヤーの名前を画面中央に1秒表示 | 半透明フルスクリーンオーバーレイ → `opacity` フェード |
| 中 | **カード回収飛行** | 一致時にカードがプレイヤーエリアへ飛んでいく | `transform: translate + scale` → `visibility:hidden` |
| 中 | **カード配置演出** | ゲーム開始時に1枚ずつ場に広がる | `animation-delay` を連番で設定（`0ms`, `30ms`, `60ms`…） |
| 中 | **役達成ポップアップ** | 結果画面で役名が1つずつフェードイン | `animation-delay` ＋ `opacity: 0 → 1` |
| 低 | **カードホバー浮き上がり** | マウスオーバーでわずかに浮く | `transform: translateY(-3px)` + `transition: 0.15s` |
| 低 | **残り枚数アニメーション** | 場の残り枚数が変化するときの数値切り替え | `countUp` 系のCSSまたは軽量JS |

### 8.3 サウンド（未実装）

| 音 | タイミング | 実装案 |

| カードめくり音 | カードを1枚めくった時 | Web Audio API / `<audio>` タグ |

## 9. 技術スタック

### 9.1 採用技術（確定）

| 項目 | 採用 | 理由 |
|------|------|------|
| 言語 | HTML5 / CSS3 / Vanilla JavaScript | ビルド不要、itch.ioにそのままZIPで出せる |
| 構成 | シングルHTMLファイル（`index.html`） | 依存ゼロ、配布しやすい |
| フレームワーク | **なし** | この規模では過剰。まず完成させることが優先 |
| ビルドツール | **なし** | 不要 |
| ランタイム | ブラウザのみ | インストール不要でプレイ可能 |

> Next.js・React等のフレームワークはこの規模では重く、itch.ioへのHTML5デプロイとも相性が悪いため採用しない。

### 9.2 将来の移行候補（機能拡張時のみ検討）

| 条件 | 候補スタック |
|------|------------|
| コンポーネント分割が必要になった場合 | Vite + Vanilla JS（フレームワークなし） |
| オンライン対戦を実装する場合 | Supabase Realtime / Socket.io |
| モバイルアプリ化する場合 | Capacitor（既存HTMLをラップ） |

## 10. リリース戦略・デプロイ

### 10.1 二段構えの展開方針

```
自前サイト（Netlify / Vercel）
  └── 作品紹介・制作意図・ポートフォリオ
```

**Netlify 最短手順：**
1. [netlify.com](https://www.netlify.com/) でアカウント作成
2. `index.html` を含むフォルダをブラウザにドラッグ&ドロップ
3. URLが即時発行（例：`https://hanafuda-xxx.netlify.app`）

### 10.3 itch.io 公開

**目的**：ゲームを実際に遊んでもらう。インディーゲームコミュニティへのリーチ。

#### アップロード手順

1. [itch.io](https://itch.io) でアカウント作成 → 「Upload new project」
2. Kind of project: **HTML**
3. `index.html` が含まれるフォルダを ZIP にまとめてアップロード
   - ⚠️ `index.html` はZIPの**ルート直下**に置くこと（サブフォルダに入れない）
4. 「This file will be played in the browser」にチェック
5. Embeds サイズを設定（推奨：幅700px以上）

#### ページ最適化

| 項目 | 推奨 |
|------|------|
| カバー画像 | 315×250px（最低）または 630×500px（推奨）、比率315:250 |
| スクリーンショット | 3〜5枚 |
| 説明文 | ゲームの特徴・遊び方を簡潔に（日本語＋将来は英語併記） |
| タグ | `hanafuda`, `card-game`, `memory-game`, `japanese`, `花札` 等 |
| AI素材 | 使用有無を必ず申告（itch.ioの規約） |

> **注意**：初公開時に「Most Recent」リストの上位に表示される。
> 粗い状態で公開すると機会を無駄にするため、最低限の完成度で出すこと。

### 10.4 GitHub との連携

```
GitHub リポジトリ（公開 or 非公開）
  ├── index.html          ← ゲーム本体
  ├── REQUIREMENTS.md     ← 本書
  └── README.md           ← 作品説明（itch.io / 自前サイト用）

        ↓ pushをトリガーに
  Netlify / Vercel が自動デプロイ（自前サイト）

        ↓ リリース時に手動で
  ZIPを itch.io にアップロード
```

## 11. 収益化

### 11.1 方針

- **v1.0 は完全無料**で公開し、まずプレイヤーを集める
- 広告は**入れない**（UXを損なう、HTML5ゲームとの相性が悪い）
- itch.io の **Pay What You Want（任意支援）** を採用

### 11.2 itch.io の収益モデル

| 設定 | 内容 |
|------|------|
| 価格設定 | 「$0 or Donate」（最低価格0円、任意で支援可能） |
| 収益分配 | itch.io へのデフォルト10%（0〜100%で変更可能） |
| 平均支援額 | 約$3.68（$1〜$5が最多） |
| 支援率の目安 | 全ダウンロードの1〜3%（10,000DLで100〜300件） |

> **最大$500の支援を1人から受けた例あり**（itch.ioブログより）。

### 11.3 段階的な収益化プラン

| フェーズ | タイミング | 施策 |
|---------|-----------|------|
| フェーズ1 | リリース直後 | 無料公開 + 任意支援 |
| フェーズ2 | 反響を見て | 追加コンテンツ（追加ルール・テーマ）を有料DLCとして販売 |
| フェーズ3 | 長期運用 | 更新・追加ステージで継続的な支援を集める |

### 11.4 参考：itch.io の規模感

- 月間約3,000万訪問者（2026年現在）
- 90万件以上のゲーム・インタラクティブプロジェクト
- インディー・実験的作品を好む熱量の高いユーザー層
