## 1. プロジェクト概要

### 1.1 コンセプト

花札の伝統的なルール・役・月の体系を保持しつつ、2つのゲームモードを提供する。

- **モードA「こいこい」**：本来の花札ルール。手札と場を使った伝統的な対戦。
- **モードB「神経衰弱」**：花札×神経衰弱のオリジナルアレンジ。裏向き全48枚から記憶と役づくりで戦う。

どちらのモードも、オンライン対戦・NPC対戦・ローカルパスプレイに対応する。

### 1.2 制作意図

- 花札の魅力をデジタルで広め、知らない人でも遊べる入口を作る
- 神経衰弱モードは「ゲームの新しい可能性」のデモとしてポートフォリオに活用
- itch.io + 自前サイトの二段構えでリリース

### 1.3 対象プレイヤー

| ターゲット | 詳細 |
|-----------|------|
| メイン | 花札に興味はあるが詳しく知らない 20〜40 代 |
| サブ | 花札を知っている人が新鮮なアレンジを楽しむ |
| プレイ形式 | オンライン対戦 / NPC対戦 / ローカルパスプレイ |

## 3. ゲームモード

### 3.1 ホーム画面の構成

```
花札ホーム
├── モードA：こいこい（伝統ルール）
│     ├── ローカル 2人対戦
│     ├── NPC 対戦（難易度選択）
│     └── オンライン 2人対戦
│           └── 期間選択：3ヶ月 / 6ヶ月 / 9ヶ月 / 12ヶ月
│
└── モードB：神経衰弱（オリジナルアレンジ）
      ├── ローカル 2〜4人対戦（パスプレイ）
      ├── NPC 対戦（難易度選択）
      └── オンライン 2〜4人対戦
```

### 3.2 モードA：こいこい ルール

#### 基本ルール

| 項目 | 内容 |
|------|------|
| 人数 | 2人（オンライン・NPC・ローカル） |
| デッキ | 48枚 |
| 手配り | 各プレイヤー8枚、場に8枚、残り山札 |
| ターン | 手札から1枚出して場と合わせる → 山札をめくって場と合わせる |
| 同月マッチ | その月のカードを取得できる |
| 役完成時 | 「こいこい（継続）」か「勝負（終了）」を選択 |
| ラウンド終了 | 手札がなくなるか「勝負」を選択した時点 |
| 期間 | 3ヶ月 / 6ヶ月 / 9ヶ月 / 12ヶ月（ラウンド数） |
| 勝敗 | 全ラウンド終了後に累計点数を比較 |

#### こいこいの役（省略なし・全役実装）

| 役名 | 条件 | 点数 |
|------|------|------|
| 五光 | 光札5枚 | 15点 |
| 四光 | 雨なし光4枚 | 8点 |
| 雨四光 | 雨含む光4枚 | 7点 |
| 三光 | 雨なし光3枚 | 5点 |
| 猪鹿蝶 | 萩に猪＋紅葉に鹿＋牡丹に蝶 | 5点 |
| 赤短 | 松・梅・桜の赤短冊 | 5点 |
| 青短 | 牡丹・菊・紅葉の青短冊 | 5点 |
| 月見で一杯 | 芒に月＋菊に盃 | 5点 |
| 花見で一杯 | 桜に幕＋菊に盃 | 5点 |
| 種（5枚〜） | 種5枚以上 | 1点 + 超過1点/枚 |
| 短冊（5枚〜） | 短冊5枚以上 | 1点 + 超過1点/枚 |
| カス（10枚〜） | カス10枚以上 | 1点 + 超過1点/枚 |

> こいこいを宣言して相手に先に役を作られた場合、相手の点数が2倍になる。

### 3.3 モードB：神経衰弱 ルール（v1.0から修正）

#### 基本ルール（v1.0からの変更点）

| 項目 | v1.0 | v2.0（修正後） |
|------|------|--------------|
| マッチ時のボーナス | なし（次のプレイヤーへ） | **あり（連続してめくれる）** |
| その他 | — | 変更なし |

#### 詳細ルール

| 項目 | 内容 |
|------|------|
| 人数 | 2〜4人（ローカル / オンライン / NPC） |
| 初期配置 | 全48枚を裏向きでグリッドに並べる |
| 1ターン | 2枚めくる |
| 成功 | 同じ月 → 回収 + **もう1ターン継続** |
| 失敗 | 異なる月 → 裏に戻して次のプレイヤーへ |
| ゲーム終了 | 場の札がすべてなくなった時点 |
| 勝敗 | 集めた役のポイント合計が多い方の勝ち |

## 9. オンラインマルチプレイヤー

### 9.1 方針・技術選定

**Firebase Realtime Database** を採用する。

| 比較項目 | Firebase | Supabase | Playroom Kit |
|---------|----------|---------|--------------|
| バニラJS対応 | ◎ | ○ | ◎ |
| ビルド不要 | ◎（CDNのみ） | △ | ◎（CDNのみ） |
| 無料枠 | 十分（1GB/月） | 十分（500MB） | 十分 |
| 設定の簡単さ | ○ | ○ | ◎（最も簡単） |
| カスタム性 | ◎ | ◎ | △ |
| itch.io との相性 | ◎ | ◎ | ◎ |

> **最終採用：Firebase Realtime Database**
> - バニラJSとの相性が良く、CDNタグ1行で導入できる
> - ゲーム状態（盤面・ターン・プレイヤー情報）をリアルタイム同期
> - ルームID方式で招待リンクを共有
> - 参考：[Multiplayer Game with Firebase and Vanilla JS](https://www.coopmode.dev/articles/multiplayer-with-firebase)

### 9.2 オンラインのゲームフロー

```
ルーム作成者（ホスト）
  → Firebase に新ルーム作成
  → ルームID（6桁）を生成・表示
  → 参加者を待つ

参加者
  → ルームIDを入力
  → Firebase のルームに接続
  → ホストがスタートを押すと全員のゲームが開始

ゲーム中
  → ホストがゲーム状態をFirebaseに書き込む（正規）
  → 全プレイヤーがリアルタイムで受信・描画
  → カードクリックはFirebaseを通じてホストへ送る
```

### 9.3 Firebase のデータ構造

```json
{
  "rooms": {
    "ABC123": {
      "mode": "memory",
      "status": "playing",
      "currentPlayer": 0,
      "board": [
        { "cardId": 12, "faceUp": false, "collected": false }
      ],
      "players": [
        { "name": "田中", "collected": [0, 1, 2], "uid": "xxx" }
      ],
      "flipped": [3, 7]
    }
  }
}
```

### 9.4 オンライン時の注意事項

- **ホスト権限**：ゲーム状態の更新はホストのみが行う（不正防止）
- **再接続**：Firebase の `onDisconnect` で離脱時の処理を実装
- **タイムアウト**：30秒操作がない場合は次のプレイヤーへ自動移行
- **セキュリティ**：Firebase Rules で自分のルーム以外への書き込みを禁止

## 10. NPC（CPU対戦）

### 10.1 難易度設定

| 難易度 | 挙動 | 実装方針 |
|--------|------|---------|
| **ランダム** | 完全ランダムで2枚めくる | `Math.random()` |
| **普通** | 一度めくったカードを一定確率（60%）で記憶する | 簡易記憶配列 |
| **強い** | めくったカードを完全に記憶する。役になるカードを優先的に狙う | 完全記憶 + 役優先スコアリング |

### 10.2 強いNPCのアルゴリズム（神経衰弱）

```
1. 記憶している「既知のペア」があれば即座に選択
2. 既知ペアがなければ：
   a. 役に必要なカードを記憶していれば、そのカードを選択
   b. 記憶していない未知のカードをランダムに選択
3. 1枚目でめくったカードと同月のカードを記憶していれば確実に選択
```

### 10.3 こいこいNPCのアルゴリズム

参考リポジトリ：[AhamSammich/lets-play-koikoi](https://github.com/AhamSammich/lets-play-koikoi)

```
1. 場と手札を比較し、マッチ可能な組み合わせを列挙
2. 役完成に近いカードの組み合わせを優先
3. こいこい判断：現在の役点数 + 期待値 > リスク なら「こいこい」
```

## 11. 技術スタック

### 11.1 フロントエンド（確定）

| 項目 | 技術 | 理由 |
|------|------|------|
| 言語 | HTML5 / CSS3 / Vanilla JavaScript | ビルド不要、itch.ioに直接デプロイ可能 |
| 構成 | シングルHTMLファイル | 依存ゼロ、配布・共有が容易 |
| フレームワーク | **なし** | この規模では不要 |

### 11.2 バックエンド（オンライン機能）

| 項目 | 技術 | 理由 |
|------|------|------|
| リアルタイム通信 | **Firebase Realtime Database** | CDN1行で導入、バニラJS対応 |
| 認証 | Firebase Anonymous Auth | アカウント登録不要でプレイ可能 |
| ホスティング | Netlify（自前サイト） + itch.io | 既存方針を維持 |

### 11.3 導入方法（Firebase CDN）

